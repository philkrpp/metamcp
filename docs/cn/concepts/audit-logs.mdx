**审计日志**功能为通过 MetaMCP 进行的所有 MCP 工具调用提供全面的可见性。按 API 密钥、端点、服务器和结果状态跟踪请求，以监控使用模式、调试问题并确保安全合规。

## 概述

审计日志捕获每个 MCP 工具调用的详细信息，包括：

- **API 密钥**：用于身份验证的 API 密钥
- **端点**：处理请求的 MetaMCP 端点
- **命名空间**：请求的命名空间上下文
- **MCP 服务器**：处理工具调用的目标 MCP 服务器
- **工具**：调用的具体工具
- **状态**：成功或错误状态
- **持续时间**：请求处理时间（毫秒）
- **错误详情**：失败请求的完整错误消息

## 功能特性

### 多维度过滤

按以下条件过滤审计日志：
- **API 密钥**：查看特定 API 密钥的请求
- **命名空间**：按命名空间上下文过滤
- **状态**：仅显示成功或失败的请求
- **时间范围**：查询特定时间段内的日志

### 分组和分析

审计日志自动按以下维度分组：
- API 密钥用于使用跟踪
- 端点用于流量分析
- MCP 服务器用于性能监控
- 结果状态用于错误率分析

### 实时监控

- 处理新请求时实时更新
- 即时查看请求模式
- 快速识别错误和异常

## 使用场景

### 安全审计

跟踪哪些 API 密钥正在访问哪些 MCP 服务器和工具。识别未经授权的访问尝试或异常使用模式。

### 性能监控

监控请求持续时间以识别慢速 MCP 服务器或工具。分析流量模式以优化资源分配。

### 调试

快速识别失败的请求及其完整错误详情。跟踪跨端点、命名空间和服务器的请求流。

### 使用分析

了解哪些工具和服务器最常被使用。跟踪 API 密钥使用情况以进行计费或配额管理。
